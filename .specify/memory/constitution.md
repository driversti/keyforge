# [KeyForge] Constitution

## Core Principles

1. This is a Go project. **Never** use Java-style inheritance or deep class hierarchies.
2. Always follow idiomatic Go practices and conventions (error handling: `if err != nil`).
3. Use `struts` and `interfaces` implicitly.
4. Keep functions small and focused.
5. If my existing code looks like "Go-flavored Java," suggest a refactor during the `/plan` phase.
